Windows Packager

Skip to end of metadata
  • Page restrictions apply
  • Added by Javier Arnaiz, last edited by Javier Arnaiz on Feb 13, 2013  (view change)
Go to start of metadata

La única responsabilidad del empaquetador de Windows es la de crear .xpa listos para ser publicados en el Windows Store, permitiendo la configuración necesaria a través del DTO que recibe por parametros, es decir, este DTO debe ser sencillo de usar por el cliente, pero debe permitir configurar ciertos parametros de ¿phonegap? y de la aplicación nativa de windows phone.

Información relacionada con el empaquetador de windows 8 & 7.

Resumen de diferencias entre uno y otro: http://msdn.microsoft.com/en-us/library/windowsphone/develop/jj207030(v=vs.105).aspx

Identificador & nombre de la aplicación

Identificador: nombre del paquete (namespace) por confirmar, no viene nada claro

http://support.hockeyapp.net/discussions/questions/909-what-is-the-bundle-id-for-windows-phone

El nombre de la aplicación se identifica dentro del fichero WMAppManifest.xml dentro del nodod App en el atributo: Title. Este title debe coincidir con el title especificado en  el proceso de subida de la aplicación al windows store. El valor por defecto es el nombre del proyecto.

The Title attribute in the App element in the WMAppManifest.xml file must contain the app title. The Title attribute must not be empty. The Application title entered in Step 2 of the submission process to Store and the title displayed on the phone must be the same.

http://msdn.microsoft.com/en-us/library/windowsphone/develop/ff769509(v=vs.105).aspx#sectionToggle5

El titulo se puede internacioanlizar para que cada usuario vea el titulo de su aplicación en idioma nativo.

http://stackoverflow.com/questions/8788840/change-or-localize-app-name-for-already-published-app

Autor

A nivel legal queremos que sea Upplication

A nivel visual se cambia en: http://www.wpxbox.com/windows-phone-dev-center-allows-to-change-publisher-name-without-support/ http://msdn.microsoft.com/en-us/library/windows/apps/Hh967786.aspx

A nivel visual por aplicación: se puede especificar en AppManifest.xml, pero luego es sobreescrito por el nombre de la cuenta, no es 100% seguro pero no encuentro mas info.

Carpetas importantes a modificar

  • ApplicationIcon.png el icono que se mostrará en el menu de inicio del movil

  • SpalshScreenImagen.jpg la imagen que se muestra cuando la aplicación esta arrancando.
  • Permisos:

ID_CAP_NETWORKING

Applications with access to network services. This must be disclosed because services can incur cost when a phone is roaming.

ID_CAP_IDENTITY_DEVICE

Applications that use device-specific information such as a unique device ID, manufacturer name, or model name.

ID_CAP_IDENTITY_USER

Applications using the anonymous LiveID to uniquely identify the user in an anonymous fashion.

ID_CAP_LOCATION

Applications with access to location services.

ID_CAP_SENSORS

Applications using the Windows Phone sensors.

ID_CAP_MICROPHONE

Applications that use the microphone. The application can record without visual indication that recording is taking place.

ID_CAP_MEDIALIB

Applications that can access the media library.

ID_CAP_GAMERSERVICES

Applications that can interact with Xbox LIVE APIs. This must be disclosed due to privacy issues since data is shared with Xbox.

ID_CAP_PHONEDIALER

Applications that can place phone calls. This may happen without visual indication for the end user.

ID_CAP_PUSH_NOTIFICATION

Applications that can receive push notifications from an Internet service. This must be disclosed as usage could incur roaming charges.

ID_CAP_WEBBROWSERCOMPONENT

Applications that use the web browser component. There are security risks with scripting.
